# Repository Guidelines

## Project Overview

NYC Taxi batch data pipeline: downloads TLC Yellow Taxi trip records (3 months), cleans/validates them with pandas+pyarrow, loads into PostgreSQL, transforms with dbt, and serves analytics via Metabase. Entire pipeline runs on Docker Compose.

## Architecture & Data Flow

```
NYC TLC Parquet Files (9.5M records)
    ↓  download_data.py
data/raw/yellow_taxi/year=2024/month=0{1,2,3}/
    ↓  spark_clean.py (pandas+pyarrow, NOT Spark)
data/silver/trips/  (partitioned parquet, ~9M valid)
data/quarantine/invalid_trips/  (~493K invalid)
data/quality_report.json
    ↓  load_to_postgres.py (COPY protocol)
PostgreSQL: raw.zone_lookup, silver.trips
    ↓  dbt run (10 models, 25 tests)
PostgreSQL: analytics.* (dims, facts, marts)
    ↓  export_gold.py
data/gold/*/  (parquet exports of marts)
    ↓
Metabase dashboard (localhost:3000)
```

Entry point in Docker: `scripts/entrypoint.py` orchestrates all steps sequentially with skip-if-already-done logic.

## Development Commands

### Docker (primary)
```bash
docker-compose build pipeline        # Build pipeline image
docker-compose up -d                 # Start postgres + metabase
docker-compose up -d pipeline        # Run full pipeline
docker-compose logs -f pipeline      # Watch pipeline progress
```

### Individual steps (inside container or local)
```bash
python scripts/download_data.py     # Download raw data
python jobs/spark_clean.py          # Clean/validate data
python scripts/load_to_postgres.py  # Load to PostgreSQL
python scripts/export_gold.py       # Export gold parquet
cd dbt_project && dbt deps && dbt run && dbt test
```

### Query PostgreSQL
```bash
docker exec nyc_taxi_postgres psql -U nyc_user -d nyc_taxi -c "SELECT ..."
```

## Code Conventions & Common Patterns

- **Language**: Python 3.12, SQL (dbt Jinja), YAML config
- **Processing engine**: pandas + pyarrow (NOT PySpark — JDK 21 incompatible with Hadoop)
- **Data loading**: PostgreSQL `COPY FROM STDIN` protocol via psycopg2 for bulk inserts (10-50x faster than `to_sql`)
- **NULL handling**: CSV export with `na_rep=""` + COPY `NULL ''` — never `\N`
- **Environment variables**: All DB config via `os.environ.get()` with fallback defaults
- **dbt profiles**: Use `{{ env_var('POSTGRES_HOST') }}` for portability
- **Entry point**: `scripts/entrypoint.py` runs steps sequentially, skips completed steps
- **Naming**: Files use `snake_case`, SQL models use `snake_case`, directories use `snake_case`

### PostgreSQL COPY Loading Pattern
```python
# Correct: CSV + COPY with empty string NULL
chunk.to_csv(buffer, index=False, header=True, na_rep="")
cur.copy_expert(f"COPY table (cols) FROM STDIN WITH (FORMAT csv, HEADER true, NULL '')", buffer)
```

### dbt Schema Convention
- Profile `schema: analytics` → all models land in `analytics` schema
- Staging = views, Marts = tables
- Sources defined in `staging/schema.yml` (silver.trips, raw.zone_lookup)

## Runtime/Tooling Preferences

- **Python**: 3.12 (slim base image)
- **Package manager**: pip (no Poetry/Hatch)
- **Key constraint**: `dbt-postgres==1.9.*` (NOT 2.0 — dbt Fusion doesn't support postgres)
- **PostgreSQL**: 16-alpine
- **Metabase**: latest image, H2 internal DB
- **Network**: Docker Compose default network, services connect by container name (`postgres`, not `localhost`)
- **Volumes**: `data/` mounted into pipeline container; `pgdata` for PostgreSQL persistence

## Data Quality Rules (applied in spark_clean.py)

| # | Rule | Implementation |
|---|------|----------------|
| 1 | pickup_time_not_null | `df["tpep_pickup_datetime"].isna()` |
| 2 | dropoff_time_not_null | `df["tpep_dropoff_datetime"].isna()` |
| 3 | trip_duration_positive | `dropoff_dt <= pickup_dt` |
| 4 | trip_distance_gt_zero | `df["trip_distance"] <= 0` |
| 5 | fare_amount_gte_zero | `df["fare_amount"] < 0` |
| 6 | total_amount_gte_fare | `df["total_amount"] < df["fare_amount"]` |
| 7 | passenger_count_1_to_6 | `df["passenger_count"] < 1 \| > 6` |
| 8 | pickup_location_valid | `df["pickup_zone"].isna()` (after left join) |
| 9 | dropoff_location_valid | `df["dropoff_zone"].isna()` (after left join) |
| 10 | duplicate_trip_id | `pd.util.hash_pandas_object()` + `drop_duplicates()` |

Invalid records → `data/quarantine/invalid_trips/` with `quarantine_reason` column.

## dbt Model DAG

```
source: silver.trips → stg_trips → fact_trips → mart_revenue_by_day
                                                → mart_revenue_by_zone
                                                → mart_trips_by_hour
                                                → mart_payment_type_summary
source: raw.zone_lookup → stg_zones → dim_zone
(stg_trips) → dim_date
(stg_trips) → dim_payment_type
```

- **Staging** (`+materialized: view`): renames columns, maps to friendly names
- **Dimensions** (`+materialized: table`): zone, date (75 rows from date range), payment_type (5 types)
- **Fact** (`+materialized: table`): ~9M rows, adds `has_tip` and `tip_percentage` derived columns
- **Marts** (`+materialized: table`): aggregated summaries for analytics
- PostgreSQL `round()` requires `::numeric` cast — `round(double precision, integer)` doesn't exist

## Testing & QA

- **dbt tests**: 25 tests (unique, not_null on key columns across all models) — run with `dbt test`
- **Data quality**: `data/quality_report.json` generated by `spark_clean.py` with input/valid/invalid counts, null counts, checks applied
- **No unit tests**: Pipeline is end-to-end validation via dbt tests + data quality report
- **Verification**: Query PostgreSQL directly or use Metabase to inspect analytics tables
